# Deploying SalaryExit India (Vercel Hobby)

This app is a standard Next.js App Router project. The Hobby plan is sufficient for static + server-rendered routes at modest traffic.

## 1. Prerequisites

- A [Vercel](https://vercel.com) account
- This repository pushed to GitHub, GitLab, or Bitbucket (or deploy via Vercel CLI)

## 2. Create the project

1. In Vercel: **Add New → Project** and import the repo.
2. **Framework Preset:** Next.js (auto-detected).
3. **Root directory:** repository root (unless you use a monorepo).
4. **Build command:** `npm run build` (default).
5. **Output:** Next.js default (no static export required).

## 3. Environment variables

Copy `.env.example` to `.env.local` for local development. In Vercel, add the same keys under **Project → Settings → Environment Variables**.

| Variable | Notes |
|----------|--------|
| `NEXT_PUBLIC_SITE_URL` | Production URL **without** trailing slash, e.g. `https://www.example.com`. Drives `metadataBase`, sitemap, robots `host`, and canonical URLs. |
| `NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION` | Search Console HTML tag verification string. |
| `NEXT_PUBLIC_GA_MEASUREMENT_ID` | Optional GA4 ID when you enable analytics. |
| `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` | Optional Plausible domain. |
| `NEXT_PUBLIC_ADSENSE_CLIENT_ID` | Optional; set **after** AdSense approval. Do not commit real IDs in the repo. |
| `NEXT_PUBLIC_ENABLE_AD_SLOTS` | `true` (default) shows placeholder slots; `false` hides them. |
| `NEXT_PUBLIC_ENABLE_CONSENT_BANNER` | `true` shows the placeholder banner; wire a CMP before real consent in EU/UK. |
| `NEXT_PUBLIC_CONTACT_EMAIL` | Email displayed on `/contact`. |

Redeploy after changing env vars.

## 4. Custom domain

1. Vercel **Project → Settings → Domains**.
2. Add your apex (`example.com`) and/or `www` subdomain.
3. Follow Vercel’s DNS instructions (A/CNAME records). Wait for DNS propagation.
4. Set **`NEXT_PUBLIC_SITE_URL`** to your **preferred** public URL (usually `https://www.example.com` or apex, consistently).
5. In Vercel, optionally redirect apex ↔ www so only one canonical host serves the site.

## 5. Google Search Console

1. Add a **URL prefix** or **Domain** property for your live site.
2. Use the **HTML tag** verification method and paste the **content** value into `NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION`, redeploy, then verify in Search Console.
3. Submit **`https://your-domain/sitemap.xml`** (generated by `app/sitemap.ts`) under **Sitemaps**.

**Verify it worked:** in Search Console, open **Sitemaps** and confirm the submitted URL shows “Success” (may take a short time). Under **Pages** / **Indexing**, confirm the property is collecting data after go-live.

## 6. Google AdSense (after approval)

1. Complete site review and policy checks in AdSense.
2. Set **`NEXT_PUBLIC_ADSENSE_CLIENT_ID`** to your `ca-pub-…` client ID so `ThirdPartyRootScripts` can load `adsbygoogle.js` (see `components/ads/ThirdPartyRootScripts.tsx`).
3. For **Auto Ads** or extra snippets Google provides in the dashboard, add them in one place only—typically next to the existing script placeholders in `app/layout.tsx` via the same client component, or exactly as Google documents for your integration path. Avoid duplicating the loader script.
4. Replace dashed **AdSlot** placeholders with real ad units when you map slots to AdSense ad units (or keep Auto Ads and use slots as reserved layout regions).

## 7. Analytics choice

- **GA4:** set `NEXT_PUBLIC_GA_MEASUREMENT_ID`.
- **Plausible:** set `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` (and leave GA empty unless you intentionally use both).

Third-party scripts load **only** when `ThirdPartyRootScripts` has a matching env var and consent allows analytics (`components/ads/ThirdPartyRootScripts.tsx`). Custom events (calculator use, share, offer compare, etc.) are documented in **`docs/analytics.md`**.

**Verify pageviews:** after deploy, open the live site (incognito helps), then check GA4 **Realtime** or the Plausible dashboard for your domain.

Update your **Privacy policy** if you change what you collect.

## 8. Local production check

```bash
npm run build
npm run start
```

Visit `http://localhost:3000` and confirm `/sitemap.xml`, `/robots.txt`, and `/manifest.webmanifest` resolve.
